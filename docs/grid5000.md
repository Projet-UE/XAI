# Grid'5000 execution guide

This project is designed so that code is versioned in Git and executed on Grid'5000 only.

## 1. Start a Grenoble frontend session

Open `https://intranet.grid5000.fr/notebooks` and launch a JupyterLab frontend with:

- `site=grenoble`
- `type=front`
- `project=irit`
- `queue=auto`

Use this session for:

- cloning or pulling the repository
- configuring GitHub authentication
- preparing the Python environment
- downloading the Kaggle dataset
- running smoke tests

## 2. Clone the repository on Grenoble

```bash
git config --global credential.https://github.com.useHttpPath true
git config --global user.name "Your GitHub Name"
git config --global user.email "your-email@example.com"

cd "$HOME"
git clone https://github.com/Projet-UE/XAI.git
cd XAI
git checkout feature/classification-xai-baseline
git pull --ff-only
```

Pushes are done with HTTPS + token from the Grenoble session.

## 3. Prepare the environment

```bash
cd "$HOME/XAI"
./scripts/grid5000_setup.sh
source .venv/bin/activate
```

The environment uses `pip install -e ".[dev]"`, so code changes in the repo are immediately reflected.

## 4. Download the Kaggle dataset on Grenoble

Expected Kaggle credentials:

- `~/.kaggle/kaggle.json`

Download the dataset:

```bash
mkdir -p "$HOME/data"
kaggle datasets download \
  -d navoneel/brain-mri-images-for-brain-tumor-detection \
  -p "$HOME/data"
unzip "$HOME/data/brain-mri-images-for-brain-tumor-detection.zip" -d "$HOME/data/raw-brain-mri"
```

Normalize the extracted content so the final layout is:

```text
$HOME/data/brain-mri-images/
├── no/
└── yes/
```

The repo scripts expect the final dataset root to be passed as:

```bash
--data-root "$HOME/data/brain-mri-images"
```

## 5. Run frontend smoke checks

These checks are intended for the Grenoble frontend session:

```bash
cd "$HOME/XAI"
source .venv/bin/activate
pytest -q tests/test_data.py tests/test_pipeline.py
python scripts/train.py --help
python scripts/evaluate.py --help
python scripts/generate_xai.py --help
```

## 6. Start a Grenoble GPU session

For training and XAI generation, launch a reserved-node Jupyter session with:

- `site=grenoble`
- `type=node`
- `project=irit`
- `queue=abaca`
- `oarres=/host=1/gpu=1`
- `walltime=04:00:00`

Keep using the same Grenoble paths:

- repo: `$HOME/XAI`
- dataset: `$HOME/data/brain-mri-images`
- artifacts: `$HOME/XAI/artifacts`

## 7. Train

```bash
cd "$HOME/XAI"
source .venv/bin/activate
python scripts/train.py \
  --data-root "$HOME/data/brain-mri-images" \
  --artifacts-dir artifacts \
  --epochs 5 \
  --batch-size 8 \
  --image-size 224
```

Outputs:

- `artifacts/splits/brain_mri_split.json`
- `artifacts/training/history.json`
- `artifacts/training/checkpoints/best.pt`

## 8. Evaluate

```bash
cd "$HOME/XAI"
source .venv/bin/activate
python scripts/evaluate.py \
  --data-root "$HOME/data/brain-mri-images" \
  --manifest-path artifacts/splits/brain_mri_split.json \
  --checkpoint-path artifacts/training/checkpoints/best.pt \
  --output-dir artifacts/evaluation
```

## 9. Generate XAI outputs

```bash
cd "$HOME/XAI"
source .venv/bin/activate
python scripts/generate_xai.py \
  --data-root "$HOME/data/brain-mri-images" \
  --manifest-path artifacts/splits/brain_mri_split.json \
  --checkpoint-path artifacts/training/checkpoints/best.pt \
  --output-dir artifacts/xai \
  --methods gradcam integrated_gradients occlusion \
  --max-samples-per-class 2
```

## 10. Keep Git clean

Do not commit:

- raw datasets
- large checkpoints
- full `artifacts/` folders

Commit only code, lightweight configs, small exported figures if needed, and documentation updates.

## 11. Publish a lightweight run snapshot

If you want a concrete experiment trace on GitHub without committing bulky outputs, create a tracked folder under `results/` with:

- a short README
- the main metrics as JSON
- the run configuration
- a compact summary of selected XAI samples

Example:

```text
results/
└── grenoble_gpu_20260324/
    ├── README.md
    ├── metrics.json
    ├── run_config.json
    └── xai_samples.json
```

This keeps the repository informative while leaving the full `artifacts/` tree on Grid'5000.
