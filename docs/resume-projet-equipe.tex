\section{Ce qu'il faut retenir}

Le dépôt contient deux pistes complémentaires:

\begin{itemize}
  \item \textbf{la piste principale}: segmentation autoPET FDG avec \texttt{nnUNet} et XAI;
  \item \textbf{la piste de secours}: classification Brain MRI avec XAI.
\end{itemize}

Le point important pour l'\^equipe est simple:

\begin{itemize}
  \item on ne garde pas tout le dataset brut;
  \item on ne relance pas des entra\^\i{}nements lourds pour rien;
  \item on s'appuie sur les r\'esultats d\'ej\`a produits et trac\'es dans \texttt{results/}.
\end{itemize}

La ligne autoPET est la plus utile scientifiquement. La ligne Brain MRI reste
utile comme baseline de secours et comme comparaison plus simple.

\section{Partie principale: autoPET FDG}

Cette partie est la plus importante pour le projet, car elle colle le mieux au
sujet initial: imagerie m\'edicale, segmentation et explicabilit\'e.

\subsection{Ce qui a \'et\'e gard\'e}

\begin{tabular}{L{5.2cm}L{9.2cm}}
\toprule
\textbf{El\'ement} & \textbf{Choix retenu} \\
\midrule
Famille de donn\'ees & \texttt{autoPET I/II} \\
Donn\'ee utilis\'ee en pratique & \texttt{FDG-PET-CT-Lesions} \\
T\^ache & segmentation de l\'esions \\
Mod\`ele de base & \texttt{nnUNet v2 (3d\_fullres)} \\
Variante principale & \texttt{post\_best\_dice\_50epochs} \\

On a volontairement travaill\'e sur un \textbf{sous-ensemble contr\^ol\'e} plut\^ot que
sur tout le volume brut. Cela rend l'exp\'erience faisable et lisible dans le
temps du projet.

\subsection{R\'esultats \`a citer}

\begin{tabular}{L{5.0cm}L{3.2cm}L{3.2cm}L{3.2cm}}

Lecture simple:

\begin{itemize}
  \item \texttt{post\_best\_dice\_50epochs} est le meilleur r\'esultat \`a montrer;
  \item \texttt{post\_low\_fp\_50epochs} sert \`a discuter le compromis faux positifs / Dice.
\end{itemize}

\subsection{Ce que la XAI veut dire ici}

Dans ce projet, la XAI ne veut pas dire ``zone rouge = cancer''. Elle veut dire:

\begin{itemize}
  \item \textbf{quelles zones ont le plus influenc\'e la pr\'ediction du mod\`ele};
  \item \textbf{pourquoi} le mod\`ele a bien fonctionn\'e ou s'est tromp\'e;
  \item si le mod\`ele regarde des zones coh\'erentes avec la l\'esion ou s'il se
  laisse attirer par d'autres zones.
\end{itemize}

L'analyse XAI la plus utile pour le rapport est celle sur les \textbf{7 cas de revue}
du r\'esultat principal:

\begin{itemize}
  \item 3 cas positifs d\'etect\'es;
  \item 2 faux positifs;
  \item 2 vrais n\'egatifs.
\end{itemize}

La m\'ethode la plus parlante pour la discussion est \texttt{Integrated Gradients}.
Sur cette s\'erie:

\begin{itemize}
  \item l'attribution est environ \textbf{4.58x} plus forte \`a l'int\'erieur de la
  l\'esion que dehors sur les cas positifs;
  \item sur les faux positifs, l'attribution est environ \textbf{5.68x} plus forte
  dans la zone pr\'edite que dans le reste de l'image.
\end{itemize}

\subsection{Pourquoi cette partie reste utile}

La piste Brain MRI n'est pas la piste principale, mais elle reste utile:

\begin{itemize}
  \item comme baseline de secours si la partie autoPET devait poser probl\`eme;
  \item comme exemple plus simple pour expliquer la XAI \`a l'\^equipe;
  \item comme r\'esultat de comparaison dans le rapport.
\end{itemize}

\section{Code utile}

Les scripts \`a conna\^\i{}tre vraiment sont:

\begin{itemize}
  \item \texttt{scripts/autopet\_prepare\_fdg.py}: pr\'epare le sous-ensemble FDG;
  \item \texttt{scripts/autopet\_train\_nnunet.py}: lance l'entra\^\i{}nement \texttt{nnUNet};
  \item \texttt{scripts/autopet\_predict\_nnunet.py}: produit les m\'etriques de segmentation;
  \item \texttt{scripts/autopet\_generate\_xai.py}: exporte les figures XAI PET/CT;
  \item \texttt{scripts/autopet\_export\_results.py}: cr\'ee les snapshots l\'egers pour Git;
  \item \texttt{scripts/generate\_xai.py}: exporte la XAI pour la classification Brain MRI.
\end{itemize}

Ce sont les scripts qui servent r\'eellement au projet. Le reste est soit de
l'historique, soit du support, soit du r\'esultat d'exploration.

\section{Ce qu'on peut ignorer maintenant}

\begin{itemize}
  \item le dataset complet \`a \~{}400 GB;
  \item les nouvelles exp\'eriences lourdes sans demande explicite;
  \item le multi-traceur / PSMA tant que le FDG n'est pas la cible directe;
  \item les gros artefacts non trac\'es dans \texttt{results/}.
\end{itemize}

\section{Conclusion}

Pour l'\^equipe, le message simple est le suivant:

\begin{itemize}
  \item garder la piste autoPET FDG comme r\'esultat principal;
  \item garder la classification Brain MRI comme backup;
  \item utiliser les figures et m\'etriques trac\'ees dans Git pour le rapport;
  \item ne pas repartir sur des exp\'eriences lourdes inutiles.
\end{itemize}
